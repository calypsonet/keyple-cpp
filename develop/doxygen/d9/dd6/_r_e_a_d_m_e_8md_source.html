17 ## Global Architecture of Keyple
18 
19 ![global architecture](docs/img/Keyple-components.svg "keyple SDK global architecture")
20 
21 The API is currently divided in two major layers:
22 - The ‘Keyple Core' : a Secure Element Proxy API which allows managing SE readers in a generic way, whaterver the reader driver or environment, and for standalone or distributed solution ([Keyple Core User Guide](./docs/KeypleCore_UserGuide.md)).
23 - A ‘Calypso Keyple extension' : a high level Calypso Processing API allowing to operate commands with a Calypso Portable Object, and to manage a secure Calypso transaction ([Keyple Calypso User Guide](./docs/KeypleCalypso_UserGuide.md)).
24 
25 Dedicated reader’s plugins have to be implemented in order to interface the SE Proxy API with the specific reader’s drivers.
26

77 ### Cloning this project
78 Examples provided in this project relies on symbolic links to handle their common dependencies. (Please refer to this [file](example/README.md) for more information).
79 
80 Although symlink support should be provided out of the box for Unix users, **Windows users** should be aware that the git option `core.symlinks` needs to be enabled before [cloning](https://help.github.com/en/articles/cloning-a-repository) this repo. Several solutions can be considered:
81 - When installing git for Windows, an option `Enable symbolic links` can be choosen. If it has not been enabled and you want to set it via the installer, a reinstallation is needed
82 - If you do not want to reinstall git, this option can be enabled afterward via the command line `git config core.symlinks true`
83 - Also, the option can be enabled once only for this specific cloning operation with `git clone -c core.symlinks=true REPO_URL`
84 
85 It is important to note that for this option to be actually working, the Windows user needs to have the **_SeCreateSymbolicLink_ permission**: a user with admin rights is typically granted with this permission.

91 ### C++ components

93 #### Prerequisites
94 Here are the prerequisites to build the keyple components (dynamic libraries)
95 - G++ 6 and higher, MSVC++ 14 (other compilers untested)
96 - libpcsclite1 package installed (Linux) to build the PC/SC plugin
97 
98 #### Windows, Linux or Macos
99 On Linux and macOS, the following command will build all the artifacts at once
100 ```
101 mkdir build; cd build; cmake .. && make
102 ```
103 
104 If using Windows or IDEs, CMake support must be installed. Compilation should automatically be handled.
